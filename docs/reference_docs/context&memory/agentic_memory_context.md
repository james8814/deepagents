当我让Agent自主管理上下文
受到Agentic Memory这篇论文的启发，我开发了一个实验性项目Memci，用于验证Agent自主管理上下文的可行性。Memci的主要设计目标是实现用户视角无限制的上下文窗口大小，以及抛弃会话概念，让用户可以像和人聊天一样，不需要通过手动开启会话来隔离上下文。
Agentic Memory: Learning Uniffed Long-Term and Short-Term MemoryManagement for Large Language Model Agents

https://arxiv.org/pdf/2601.01885
要实现上述的设计目标，本质上就是根据当前用户的话题，将相关的记忆放入上下文，而将其他的上下文清除。
Memci项目将上下文窗口组织为一棵类似目录树的Page树结构，系统自动将对话详细内容放入DetailPage（类比文件），而多个相关Page则可以被合并为一个ContentsPag（类比目录），ContentsPage并没有实际内容，而是代表一组相关Page的一个摘要。所有的Page都有状态，只有Active状态的Page对Agent来说是可见的。


通过Page树的抽象，Agent的“大脑“中维护了一个类似文件系统的层级结构。Agent可以通过类似文件系统的 mkdir mv rm 等工具来维护这棵Page树。我还提供了两个特殊的tool：hide和expand，用来控制Page的可见性。
通过这套树形的context抽象，理论上可以让Agent维护远大于实际上下文窗口的上下文空间。Agent根据用户的prompt动态地在Page树中执行hide，expand操作，就可以将与当前prompt相关的上下文展示，而隐藏无关的上下文。Agent可以灵活地create page来记录一些小笔记，放入Page树中来影响自己之后的状态
这套Page树抽象似乎有某种美感，但当我真正实现出来后，发现了很多的问题。
管理这套上下文系统给Agent带来了太多的非任务负担。有趣的是，这件事是Memci自己告诉我的；我让它评价我它设计的这套系统怎么样，它指出它一边需要完成我交代的任务，一边需要时时思考“应该把哪些page hide，把哪些expand，哪些page归类不合适，需要重新整理”，这让它无法专注地完成当下的核心任务。Agentic memory论文中的系统是通过RL学习memory管理策略的，同时他们的记忆系统比我设计的简单很多，而我这套复杂系统+prompt，的确让Agent的响应速度显著变慢了。
使用树形结构管理所有上下文太过昂贵。大量对话信息其实很难被严格组织成树形结构，Agent在上下文窗口耗尽之前几乎不会主动去将几轮对话信息合并为一个目录，而是采取隐藏的策略。对话信息并不是经过提取后高密度的信息，强行让Agent组织为一科严格的树结构其实并不必要。

我问Memci有没有自发的动因
Memci项目还在推进中，但我陷入了停滞，因为Memci正处在很多Agent项目共有的困境中：能用，但是不好用。并且是不知从何改进的不好用。我现在迫切需要的是确立一个优化方向，一个Memci不同于其他的Memory项目的地方。这需要先搞清楚Memory究竟要给Agent带来什么。
我意识到memory和skill存在一些联系。我们看很多文章，都介绍说skill允许我们给agent定义一套工作流。那为什么需要skill呢，因为大模型本质上是个概率生成器，它是具备随机性的，那我们要通过大模型去驱动Agent完成任务就必然存在随机性，那对于一些经常需要复用的环节，我们要确保Agent稳定运行，就必须要最小化这种随机性，那就是让Agent学会一套固定的工作流程，以后按照这套流程来就不容易出错了。所以说，skill是通过文档来持久化经验，利用大模型上下文学习能力来让它获得类似持续学习的能力。
Memory和skill的关系在这里引出来了。我认为，memory的最大价值不在于存储和检索，而在于让Agent具备持续演化，持续学习的能力。现在Agent很强大，但使用体验总是还举例理想有一定的距离，我把这段距离称为摩擦感。这种摩擦感举个例子就好理解了，和Agent对话用中文，但经过一次上下文压缩就说回英文了；在A会话要求Agent调整一下语气，上下文窗口一长，或者换到B会话，就不能再生效了。我们需要的是一个好学的实习生，虽然不聪明，但会总结，会进步，我们可以通过自然语言去持久化地微调一些行为。更激进一点的，能不能通过自然语言的方式去教会Agent某个领域的知识？例如不通过SFT或RL，不修改系统提示词，通过对话的方式把Agent调成一个懂得高中数学领域知识，懂得高中数学解题步骤的专业Agent？
Skill就提供了这种沉淀领域经验的作用，memory的存储和检索功能可以用来沉淀领域知识，那如果具备memory的Agent能够自主从错误、矛盾、用户反馈中总结出领域经验呢？
是的，让具有memory的Agent去自主地生成skill来沉淀经验，或许是一种模拟持续学习的路径。
