2️⃣  快速事实
================================================================================

需求问题:
  ❌ SubAgent 执行完后，主 Agent 无法看到工具调用历史
  ❌ 开发者无法理解 SubAgent 如何解决问题
  ❌ 无法调试失败的子任务
  ❌ 无法验证工具调用的正确性

提议解决:
  ✅ 新增 subagent_logs 状态字段记录工具调用历史
  ✅ 自动脱敏敏感数据（tokens, passwords, API keys）
  ✅ 自动截断大输出防止状态爆炸
  ✅ 完全向后兼容，100% 可选

实现成本:
  💻 代码: ~100 行核心代码 + ~50 行状态 + ~150 行测试
  ⏱️  时间: 4-5 个工程日
  🚀 复杂度: 低（中间件模式）
  💰 风险: 无（不修改现有代码路径）

4️⃣  风险评估与缓解
================================================================================

识别的风险:
  1. 性能开销      → 缓解: 日志采用 append，不影响核心路径
  2. 内存爆炸      → 缓解: 自动截断 + 日志限制
  3. 敏感数据泄露  → 缓解: 自动脱敏 token/secret/password
  4. 兼容性问题    → 缓解: NotRequired 字段，不影响现有代码

所有风险都可控，无阻止因素。
